Logical Operators

1. AND Operator

  • The AND operator is used to combine multiple media features in a media query, where all conditions must be true for the styles to apply.

  • For example, if you want to apply styles when the viewport is both a minimum width and in landscape orientation

2. NOT Operator

  • The NOT operator negates a media query. It's used to apply styles when the query's conditions are not met.

  • This can be useful for applying styles to all devices except those that meet a certain condition.

3. Commas (Implied OR)

  • In CSS media queries, a comma is used to separate multiple media queries, which acts like an OR operator.

  • Each media query is evaluated independently, and if any of the queries returns true, the styles are applied.

Summary of Use Cases

Operator Use Cases
AND Useful for targeting a very specific range or type of devices.
OR Helpful when you want to exclude certain devices or conditions.
Commas (OR) Ideal for broader conditions where you want the styles to apply in multiple different scenarios


Media Types

1. all

  • It doesn't specify any particular device type, so the styles will apply universally across print, screen, speech, and other media types unless overridden by more specific media queries.

  • It's the default media type if none is specified.

2. print

  • This media type targets print devices or documents viewed in print preview mode.

  • Styles defined with print are used when the document is printed, such as hiding elements that are not relevant for print, changing color to black and white, or modifying layout to fit a page.

3. screen

  • The screen media type targets screens of computers, tablets, and smartphones.

  • This is the most common media type used for designing responsive web layouts and styles that are specific to screen viewing, like animations or hover effects.

4. speech

  • This media type is used for speech synthesizers.

  • Styles under this type can be used to control how a document is spoken.

Summary of Use Cases

Media Type Use Cases
all General styles that are applicable to all devices, like font family or color.
print Formatting documents for printing, like changing layout for a printer-friendly version or hiding non-essential elements (like navigation).
screen Responsive web design, where different styles are applied depending on the screen size or resolution (like adjusting layout for mobile devices).
speech Accessibility, where certain content is customized for screen readers, like altering the content order or hiding visual but not auditory content.


Keywords


1. min-width / max-width

  • Defines the minimum or maximum width of the viewport. Example: @media (min-width: 600px) { ... }

2. min-height / max-height

  • Similar to width, but for the height of the viewport. Example: @media (max-height: 800px) { ... }

3. orientation: portrait / landscape

  • Targets the orientation of the device (vertical or horizontal). Example: @media (orientation: landscape) { ... }

4. aspect-ratio / min-aspect-ratio / max-aspect-ratio

  • Targets the width-to-height ratio of the viewport. Example: @media (min-aspect-ratio: 16/9) { ... }

5. resolution / min-resolution / max-resolution

  • For targeting the deviceâ€™s screen resolution. Example: @media (min-resolution: 300dpi) { ... }

6. color / min-color / max-color

  • Queries the number of bits per color component of the output device. Example: @media (min-color: 8) { ... }

7. color-index / min-color-index / max-color-index

  • Relates to the number of colors the device can display. Example: @media (max-color-index: 256) { ... }

8. monochrome / min-monochrome / max-monochrome

  • For targeting monochrome devices with the number of bits per pixel. Example: @media (monochrome) { ... }

9. grid

  • Determines whether the device is a grid-based or bitmap. Example: @media (grid: 1) { ... }

10. hover / any-hover

  • Queries whether the primary input mechanism allows the user to hover over elements. Example: @media (hover: hover) { ... }

11. pointer / any-pointer

  • Determines the accuracy of the primary pointing device. Example: @media (pointer: coarse) { ... }

12. update-frequency

  • Targets devices based on how frequently the output device can modify the appearance of content. Example: @media (update-frequency: slow) { ... }

13. scripting

  • For checking if scripting (JavaScript) is available. Example: @media (scripting: none) { ... }

Syntax


    @media only|not media-type only|and|not (media-feature) {
      
      /*
      A set of CSS rules to be applied to the document
      if the media type and expression conditions are met
      */

    }


Declaration Approach

1. Nesting Style Declarations inside Media Queries

  • This approach involves writing a media query and then defining all the relevant styles inside it.

  • Example: 
            @media (min-width: 600px) {

          .container {
              width: 80%;
          }
          h1 {
              font-size: 2em;
          }
          /* more styles */

        }

2. Nesting Media Queries inside Style Declarations

  • In this approach, you define your styles for an element or class, and nest media queries within these definitions. This is particularly common in preprocessors like Sass or Less.

  • Example: 
            .container {

            width: 100%;

            @media (min-width: 600px) {
                width: 80%;
            }

        }

        h1 {

            font-size: 1.5em;

            @media (min-width: 600px) {
                font-size: 2em;
            }

            /* more styles */
        }


Summary of Use Cases


Approach Pros Cons
Nesting Style Declarations inside Media Queries
  • Easier to understand site layout on different devices
  • Keeps all styles for a specific breakpoint in one place
  • Can lead to repetition for similar styles at different breakpoints
  • Harder to track styles for a specific element across breakpoints
Nesting Media Queries inside Style Declarations
  • Easier to find all styles for a specific element or class
  • Reduces repetition by not redeclaring the selector
  • Media queries are spread out, making them harder to locate
  • May result in longer CSS files for complex layouts


Viewports

1. Visual Viewport

  • The visual viewport refers to the area of a webpage that is currently visible to the user. It's the part of the page you're actually seeing on your screen and is accordant to your device's resolution.

  • It can change size when a user zooms in or out. Zooming in decreases the size of the visual viewport (showing less of the page but in greater detail), while zooming out increases its size (showing more of the page but in less detail).

2. Layout Viewport

  • The layout viewport, on the other hand, is the area where the browser lays out the webpage before adjusting for the visual viewport.

  • It remains constant, regardless of the user's zoom level. This means that elements sized with percentages or viewport units will stay the same size relative to the layout viewport, not the visual viewport.

Layout Types

1. Responsive Web Design

  • Responsive design uses a flexible, fluid layout that adapts to the size of the user's screen.

  • This approach provides a consistent user experience across different devices, as the content reflows to fit the available space, whether on a desktop, tablet, or mobile device.

2. Adaptive Web Design

  • Adaptive design involves creating multiple fixed layouts for different screen sizes.

  • Unlike responsive design, adaptive design doesn't rely on a single changing layout but switches between different predefined layouts based on the detected screen size.


Mobile First Approach

1. min-width approach

  • A lightweight version for mobile devices is created, and all styles for wider screens are hidden inside the corresponding media queries.

2. min-width and max-width combination

  • The use of only min-width means that the elements' lower indent will need to be nullified for media query styles with min-width: 768px; and for media query styles with min-width: 1024px, we will need to nullify the frame styles.

  • In such situations, it's good practice to close specific styles in certain intervals
